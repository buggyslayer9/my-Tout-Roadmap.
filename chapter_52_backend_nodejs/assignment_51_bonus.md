# Bonus Assignment 51: Resource-Oriented Task API Design

Design a small REST-style task API before building it.

## Requirements

Define routes for:

- listing tasks
- creating a task
- getting one task
- updating a task
- deleting a task

For each route, write:

- HTTP method
- path
- short purpose

## Stretch Goals

- add user-owned tasks
- add status filtering
- add one validation rule per route
