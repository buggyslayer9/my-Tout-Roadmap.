# 44.3 Browser DevTools Basics

When learning the web, the browser is not only where your page runs.

It is also one of your most important debugging tools.

Browser DevTools help you inspect:

- HTML structure
- CSS rules
- console output
- network requests

## Why DevTools Matter

Without DevTools, beginners often guess.

With DevTools, you can actually inspect what the browser sees.

That makes debugging much faster and much more educational.

## Useful Panels

- **Elements**: inspect and edit HTML and CSS
- **Console**: see logs and errors
- **Network**: inspect requests and responses
- **Application/Storage**: inspect local storage, cookies, and related data

## Good Beginner Habits

- inspect broken layouts instead of guessing
- read console errors carefully
- use `console.log` intentionally, not everywhere forever
- check the Network panel when data does not load

## Learning Check

- Can I inspect an element and see which CSS rules apply?
- Can I find an error in the browser console?
- Can I explain why DevTools speed up learning?

## Recommended Follow-Up

Open a page you built and inspect:

- one element
- one CSS rule
- one console message

That simple exercise builds strong frontend habits early.
