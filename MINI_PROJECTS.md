# Mini Projects Roadmap

Mini projects are the bridge between reading lessons and finishing larger capstones.

Use them to:

- prove understanding quickly
- build confidence before big projects
- collect portfolio pieces gradually
- discover weak spots early

## Recommended Mini Projects By Phase

### C Foundations

- Array statistics tool
- String cleaner and formatter
- CSV-backed task tracker
- Memory-safe dynamic contact list

### GTK4 Desktop

- Counter app
- Contact form
- Notes window
- File-based text viewer

### Rust Core

- CLI expense splitter
- Borrow-checker playground
- Enum-based command parser
- Small reusable utility crate

### Systems and Linux

- File existence checker
- Mini shell
- TCP chat client/server
- Simple Linux service wrapper

### Web

- Personal landing page
- Accessible form page
- Browser game
- Small backend API
- Full-stack CRUD starter

### Mobile

- Habit tracker
- Expense tracker
- API-backed list app
- Offline notes app

### DevOps and Final Delivery

- CI-enabled sample project
- Versioned release workflow
- Deployment checklist repo
- Monitored demo backend

## Recommendation

Try to finish:

- one mini project every 2 to 4 chapters
- one milestone project every major phase
- one flagship capstone at the end

That rhythm keeps the roadmap challenging without making it feel endless.
