# 45.1 Mobile FFI: Rust inside Flutter

You have learned Rust (Phase 3) and Flutter (Phase 7). Now, prepare to unlock the ultimate "Hero" mobile development superpower: **Running Rust code directly inside your Flutter app**.

## Why use Rust in Flutter?

Dart (Flutter's language) is great for UI, but it struggles with heavy, cpu-bound tasks. If you write a media encoder, a complex physics engine, or severe data processing in Dart, your app might drop frames.

By compiling Rust to a native mobile library (`.so` for Android, `.a` for iOS) and calling it via Foreign Function Interface (FFI), you get:
1. **Insane Performance:** C-level speed directly in your app.
2. **Memory Safety:** No random C++ segfaults crashing your app.
3. **Code Sharing:** Write your core business logic ONCE in Rust, then use it in your Axum Backend, your React WebWasm frontend, AND your iOS/Android Flutter app!

## The Magic of `flutter_rust_bridge`

Writing manual FFI bindings (C headers, unsafe pointers) for mobile is painful. The open-source `flutter_rust_bridge` (FRB) library completely automates this.

It generates the Dart code and the Rust C-bindings automatically. You just write normal Rust, and call it like normal Dart.

### 1. Setup

Install the CLI:
```bash
cargo install flutter_rust_bridge_codegen
```

Initialize your Flutter project to support Rust:
```bash
flutter_rust_bridge_create my_hero_app
```

### 2. The Rust Code (Backend)

Inside the generated `rust/src/api.rs`:

```rust
// This is pure, safe Rust. No `unsafe` or `extern "C"` needed!
// FRB will automatically generate the Dart equivalent.

pub struct VideoProcessor {
    pub resolution: u32,
    pub format: String,
}

impl VideoProcessor {
    pub fn new(resolution: u32) -> Self {
        Self {
            resolution,
            format: "mp4".to_string(),
        }
    }

    pub fn compress_video(&self, raw_bytes: Vec<u8>) -> Vec<u8> {
        // Run insanely fast Rust compression logic here...
        // This runs in a background thread automatically!
        println!("Compressing {} bytes of video...", raw_bytes.len());
        
        let mut compressed = raw_bytes;
        compressed.truncate(compressed.len() / 2); // Fake compression
        compressed
    }
}
```

### 3. The Dart Code (Frontend)

FRB automatically generates the Dart classes. You simply import them and use them in your Flutter UI!

```dart
import 'package:flutter/material.dart';
import 'package:my_hero_app/rust/api.dart'; // Auto-generated by FRB

class VideoScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: () async {
        // 1. Create the Rust object (instantiated in Rust memory)
        final processor = await VideoProcessor.new(resolution: 1080);
        
        final fakeData = List.filled(1000000, 0); // 1MB
        
        // 2. Call the Rust method!
        // This is strictly typed, asynchronous, and does NOT block the UI thread!
        final compressed = await processor.compressVideo(rawBytes: fakeData);
        
        print('Compressed size: ${compressed.length}');
      },
      child: Text('Compress Video (Powered by Rust 🦀)'),
    );
  }
}
```

This architecture is used by massive companies (like 1Password) to share their core cryptographic code across every platform flawlessly.
